# Contributing

![Software License: MPL-2.0](https://img.shields.io/badge/Software_License-MPL--2.0-slateblue?style=flat)

Thank you for wanting to contribute to Marukyu Marble!

## Before You Start

- For large changes (new features, architecture changes), please [open an issue](https://github.com/lsy7851/Wplace-MarukyuMarble/issues/new/choose) first to discuss. This saves your time if the direction doesn't align with the project. Small fixes (typos, minor bugs) can go straight to a PR.
- If you're new to JavaScript, check out this [roadmap for learning JavaScript](https://roadmap.sh/javascript).

## Guidelines

- Follow the existing code style and file structure.
- One feature per PR — don't bundle unrelated changes together.
- Write clear commit messages that explain what changed and why. "Added some things" is not a good commit message.
- Add comments where the logic isn't obvious. We may ask for clarification if the code is hard to follow.
- Maintain proper attribution and respect upstream contributors.

## How To Contribute

1. Read these guidelines.
2. [Open an issue](https://github.com/lsy7851/Wplace-MarukyuMarble/issues/new/choose) if your change is non-trivial.
3. Fork the repository and set up the [development environment](./BUILDING.md).
4. Create a feature branch and make your changes.
5. Submit a pull request.

## Project Goals

The goal of this project is to provide an extensible, high-performance open-source template overlay for pixel canvas sites.

- Clean, maintainable codebase that is easy to extend with new features.
- Efficient rendering and caching for smooth performance.
- Open-source and unobfuscated.
